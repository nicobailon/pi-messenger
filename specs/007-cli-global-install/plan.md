# Plan: CLI Global Install via jiti Wrapper

## Summary

Make `pi-messenger-cli` globally available by writing a shell wrapper to `~/.pi/agent/bin/` during `install.mjs`. The wrapper invokes pi's built-in jiti to run the CLI's TypeScript source directly from the extension directory. Zero new dependencies, zero bundling, changes are immediately live after extension updates.

## Approach

### What changes

**1. install.mjs — Constants (line ~24)**

Add:
```js
const BIN_DIR = path.join(os.homedir(), ".pi", "agent", "bin");
const CLI_WRAPPER_PATH = path.join(BIN_DIR, "pi-messenger-cli");
```

**2. install.mjs — Remove section (line ~108)**

Unconditional wrapper cleanup before extension removal:
```js
if (isRemove) {
  if (fs.existsSync(CLI_WRAPPER_PATH)) {
    fs.unlinkSync(CLI_WRAPPER_PATH);
    console.log("Removed CLI: " + CLI_WRAPPER_PATH);
  }
  if (fs.existsSync(EXTENSION_DIR)) {
    fs.rmSync(EXTENSION_DIR, { recursive: true });
    console.log("Removed pi-messenger from " + EXTENSION_DIR);
  } else {
    console.log("pi-messenger extension not installed");
  }
  process.exit(0);
}
```

**3. install.mjs — Collision guard comment (line ~147)**

Add comment explaining why collision guard also skips wrapper:
```js
// Collision guard skips both extension copy AND CLI wrapper creation.
// Dev workflow uses tsx from the repo directly — wrapper is for end-user installs only.
```

**4. install.mjs — Wrapper creation (after copyDir, end of file)**

Resolve jiti via `npm prefix -g` (not `which pi` — avoids hard dependency on pi being in PATH at install time):

```js
// ─── CLI wrapper ──────────────────────────────────────
function resolveJitiPath() {
  try {
    const npmPrefix = execFileSync("npm", ["prefix", "-g"], { encoding: "utf-8" }).trim();
    const jitiPath = path.join(npmPrefix, "lib", "node_modules",
      "@mariozechner", "pi-coding-agent", "node_modules",
      "@mariozechner", "jiti", "lib", "jiti-cli.mjs");
    if (fs.existsSync(jitiPath)) return jitiPath;
  } catch {}
  return null;
}

const jitiPath = resolveJitiPath();
if (jitiPath) {
  fs.mkdirSync(BIN_DIR, { recursive: true });
  const wrapper = `#!/bin/bash
# Generated by pi-messenger install — do not edit
JITI_PATH="${jitiPath}"
EXT_DIR="${EXTENSION_DIR}"
if [ ! -f "$JITI_PATH" ]; then
  echo "pi-messenger-cli: jiti not found at $JITI_PATH" >&2
  echo "Pi may have been reinstalled. Re-run: npx pi-messenger" >&2
  exit 1
fi
exec node "$JITI_PATH" "$EXT_DIR/cli/index.ts" "$@"
`;
  fs.writeFileSync(CLI_WRAPPER_PATH, wrapper, { mode: 0o755 });
  console.log("CLI:      pi-messenger-cli → " + CLI_WRAPPER_PATH);
} else {
  console.log("CLI:      skipped (pi not found — install pi first, then re-run)");
}
```

**5. crew/runtime-spawn.ts — CLI validation (after line ~49)**

Add validation inside the existing `skipCommandCheck` guard:

```typescript
if (runtime !== "pi" && !options?.skipCommandCheck) {
  validateCommandAvailable(command);
  // Also verify pi-messenger-cli is reachable — workers need it for mesh ops
  try {
    execFileSync("which", ["pi-messenger-cli"], { stdio: "ignore" });
  } catch {
    warnings.push(`${runtime}: pi-messenger-cli not found in PATH — workers cannot communicate with the mesh. Run: npx pi-messenger`);
  }
}
```

This is a **warning**, not a fatal error — the worker can still function without mesh coordination. The runtime command (e.g., `claude`) is the hard dependency; the CLI is soft.

## Test Strategy

### New tests

1. **Wrapper creation test**: Run install logic → verify `~/.pi/agent/bin/pi-messenger-cli` exists, is executable, contains correct jiti path and extension dir
2. **Wrapper removal test**: Run remove logic → verify wrapper is deleted even if extension dir doesn't exist
3. **Idempotent test**: Run install twice → no errors, wrapper content is correct
4. **Graceful failure test**: Create wrapper with non-existent jiti path → run wrapper → verify exit code 1 and error message contains "Re-run: npx pi-messenger"
5. **Spawn-time CLI warning**: Mock `which pi-messenger-cli` to fail → verify warning is added to `spawnResult.warnings`

### Existing tests

All 477 existing tests must continue to pass. The `tests/crew/cli.test.ts` (17 tests) runs the CLI via `npx tsx` from the dev repo — this path is unaffected by the wrapper.
